# Политика безопасности

## Сообщение об уязвимостях

Если вы обнаружили уязвимость безопасности в проекте Отсчёт, **пожалуйста, не создавайте публичный issue**. Вместо этого отправьте отчет на адрес электронной почты мейнтейнера.

### Как сообщить об уязвимости

1. **Отправьте email** с описанием уязвимости
2. **Включите**:
   - Описание уязвимости
   - Шаги для воспроизведения (если возможно)
   - Возможное воздействие
   - Предложенное исправление (если имеется)

3. **Ожидайте ответа** в течение 48 часов

### Процесс обработки

- Мы подтвердим получение вашего отчета
- Начнём работу над исправлением
- Будем держать вас в курсе прогресса
- Выпустим исправление как можно скорее
- Дадим вам кредит в публичном уведомлении (если вы согласны)

## Лучшие практики безопасности

При использовании бота соблюдайте:

1. **Переменные окружения**
   - Никогда не коммитьте `.env` файл
   - Используйте `.env.example` для примеров
   - Храните токены в переменных окружения или менеджере секретов

2. **Базы данных**
   - Используйте надёжные пароли для PostgreSQL
   - Ограничивайте доступ к БД сетью
   - Регулярно создавайте резервные копии

3. **Токены бота**
   - Используйте разные токены для разработки и продакшена
   - Если токен скомпрометирован, немедленно создайте новый
   - Никогда не предоставляйте токен третьим лицам

4. **Зависимости**
   - Регулярно обновляйте зависимости
   - Следите за безопасностью пакетов через GitHub Security Alerts
   - Используйте pinned версии в production
