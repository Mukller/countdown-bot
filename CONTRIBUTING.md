# Руководство по внесению вклада

Спасибо за интерес к участию в проекте Отсчёт! Мы рады любому вкладу: от отчетов об ошибках до новых функций.

## 📋 Перед началом

1. Прочитайте [README.md](README.md) для понимания проекта
2. Ознакомьтесь с [Кодексом поведения](CODE_OF_CONDUCT.md)
3. Проверьте существующие [Issues](https://github.com/Mukller/countdown-bot/issues) и [Pull Requests](https://github.com/Mukller/countdown-bot/pulls)

## 🐛 Отчет об ошибках

Ошибки отслеживаются через GitHub Issues. При создании отчета об ошибке:

1. **Используйте четкий описательный заголовок**
   - Плохо: "Бот не работает"
   - Хорошо: "Календарь не отображается при выборе даты в Firefox"

2. **Предоставьте подробное описание**
   - Шаги для воспроизведения
   - Фактическое поведение
   - Ожидаемое поведение
   - Скриншоты, если применимо

3. **Укажите свою окружение**
   - ОС и версия Python
   - Версия бота
   - Версия Telegram клиента

## ✨ Предложение улучшений

Предложения улучшений отслеживаются через GitHub Issues. При предложении улучшения:

1. **Используйте четкий описательный заголовок**
2. **Предоставьте подробное описание функции**
   - Используемый случай
   - Как она должна работать
   - Возможные альтернативы

## 🔀 Pull Requests

1. **Форкните репозиторий** и создайте свою ветку из `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Установите зависимости**:
   ```bash
   pip install -r requirements.txt
   pip install -r requirements-dev.txt  # для разработки
   ```

3. **Внесите изменения**:
   - Следуйте стилю кода проекта
   - Добавьте или обновите тесты для новых функций
   - Обновите документацию, если требуется

4. **Тестируйте ваш код**:
   ```bash
   python -m pytest
   ```

5. **Закоммитьте ваши изменения**:
   - Используйте понятные сообщения коммитов
   - Одна функция/исправление на коммит

6. **Пушьте в вашу ветку**:
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Откройте Pull Request**:
   - Опишите, что вы изменили и почему
   - Привяжите связанные Issue
   - Дождитесь review

## 📝 Стиль кода

Мы следуем [PEP 8](https://www.python.org/dev/peps/pep-0008/) с некоторыми отступлениями:

- **Длина строки**: максимум 100 символов
- **Импорты**: отсортированы по группам (стандартная библиотека, сторонние, локальные)
- **Типизация**: используйте type hints где возможно
- **Комментарии**: только для сложной логики

## 🧪 Тестирование

Перед отправкой PR убедитесь, что:

1. Все существующие тесты проходят
2. Вы добавили тесты для новой функциональности
3. Покрытие кода не снизилось

```bash
# Запуск тестов
python -m pytest

# С отчетом о покрытии
python -m pytest --cov=app tests/
```
